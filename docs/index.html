        📊 Benchmark Analysis & Insights
      

      
      

        

          

            ⚠️ 1. State Tracking Bottleneck
          

          

            The baseline gemma-4-26B model frequently failed due to spatial token tracking errors. Without external tools, the model repeatedly proposed illegal moves or hallucinated that squares were open when line-of-sight pieces blocked them. This triggered extensive self-correction loops and API retries, eventually causing context timeout failures.
          

        

        
        

          

            🛡️ 2. Symbolic Grounding Benefits
          

          

            When integrated with the Queen MCP Server, the model's average retry count dropped to just 0.18 per puzzle. Because the Prolog engine automatically handles strict relational verification, the LLM no longer wastes processing cycles guessing legal move arrays. Instead, it leverages the filtered insights to focus entirely on high-level tactical pruning.
          

        

        
        

          

            ⚡ 3. Efficient MoE Architectures
          

          

            These results show that highly optimized Mixture-of-Experts (MoE) models do not need to scale up their raw parameter sizes to tackle complex logical boundaries. Grounding the system with a dedicated, symbolic logic referee allows the model to perform at a tier typically reserved for significantly larger frontier models.
